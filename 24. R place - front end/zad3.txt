GIT:
Git - софтуер за version control

Един проект го пазим в едно repository.

Помощни команди за Cmd/ Powershell:
	Влиза в директория
cd име_на_директория 
	Излиза от директория
cd ..
	Съдържание на директория
dir
ls (В powershell)
	Създаване на директория
mkdir


Създаване на repository - git init

В git отделните версии на проекта ни са разделени в нещо наречено Commit. Всеки път като искаме да запазим версия на проекта ни - правим Commit.

Как се прави Commit:

1. Подготвя (stage-ва) файла за да бъде добавен в следващия Commit.
git add README.txt
git add * (добавя всички файлове)

2. Показва кои файлове са add-нати, кои файлове не са add-нати и кои файлове са commit-нати
git status

3. Създаване на самия Commit
git commit -m "Commit message"

Разглеждане на всички направени Commit-и.
git log

Зареждане на кода от последния Commit:
git checkout --



Качване на git repository в github:
1. Трябва да добавите публичен ssh ключ в github.com
- отворете cmd
- напишете ssh-keygen
- влезте там където е създаден ssh ключа (C:\Users\dimitar\.ssh)
- отворете файла ime_na_kluch.pub с Notepad(по default Името на ключа е id_rsa)
- копирайте съдържанието му в github -> Settings -> SSH AND GPG Keys -> New SSH Key

2. Да добавим remote към локалното repository:
- във CMD-то дето е в вашето git repository:
- git remote add IME_NA_REMOTE_A SSH_LINK_KUM_REMOTE (SSH_LINK_KUM_REMOTE ви го дава github като създадете ново github repository).

3. Да качим локалните commit-и към github

git push IME_NA_REMOTE_A master
